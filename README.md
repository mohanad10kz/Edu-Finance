# 💳 EduFinance — نظام إدارة الشؤون المالية التعليمية

**EduFinance** هو تطبيق سطح مكتب متكامل (Desktop Application) مطوّر بلغة **Java** وبيئة العمل **JavaFX** لإدارة الشؤون المالية في المؤسسات التعليمية (مدارس، مراكز تعليمية، إلخ). يرتكز النظام على تتبع الرسوم الدراسية للطلاب وإصدار وصولات الدفع، وإدارة رواتب المعلمين بنوعيها (الثابت والبالحصة) مع إمكانية طباعة مسيرات الرواتب وسندات الصرف بصيغة PDF باللغة العربية.

التطبيق مصمم ومبني بالكامل وفق منهجية البرمجة كائنية التوجه (**OOP**) وتطبيق أنماط التصميم البرمجية (**Design Patterns**) ليكون نموذجاً أكاديمياً وهندسياً متقدماً.

---

## 🌟 الميزات الرئيسية (Key Features)

*   **إدارة الطلاب والأقساط:** تسجيل الطلاب وتسكينهم في مراحلهم الدراسية وحساب المبالغ المدفوعة والمتبقية تلقائياً وتوليد إيصالات الدفع.
*   **إدارة المعلمين والأجور:** تسجيل المعلمين وتحديد نمط الدفع (ثابت شهرياً أو بالحصة) مع حساب مستحقاتهم ديناميكياً.
*   **توليد تقارير PDF احترافية:** طباعة سندات قبض الأقساط للطلاب وسندات صرف الرواتب للمعلمين باللغة العربية ودعم كامل لتنسيق النصوص والخطوط.
*   **أدوار المستخدمين والصلاحيات (RBAC):**
    *   **المدير (Admin):** يمتلك كامل الصلاحيات لإدارة الطلاب والمعلمين والمستندات المالية وإدارة المستخدمين والمراحل الدراسية في نافذة الإعدادات.
    *   **المستخدم العادي (Regular User):** صلاحيات محدودة تُمكّنه من تسجيل دفعات الطلاب وصرف رواتب المعلمين دون القدرة على تعديل الإعدادات أو حذف الطلاب والمعلمين.
*   **قاعدة بيانات محلية:** تخزين كافة البيانات بشكل آمن ومحلي في قاعدة بيانات **SQLite** خفيفة وسريعة التثبيت والتشغيل.

## 🛠️ التقنيات والمكتبات المستخدمة (Tech Stack)

*   **Java SE 17 (LTS):** اللغة الأساسية للتطوير بكامل ميزاتها المستقرة.
*   **JavaFX 17.0.8:** لبناء واجهات المستخدم الرسومية (GUI) بتنسيق عصري وتفاعلي.

*   **Maven:** أداة إدارة المشروع وتثبيت التبعيات وبناء النظام.

## 📐 الهندسية البرمجية ومبادئ التصميم (Architecture & Design)

يعتمد المشروع على هندسة معمارية واضحة تفصل المهام وتلتزم بأعلى معايير جودة الكود:

### 1. نمط التصميم المعماري (MVC Pattern)
يتم تقسيم المشروع إلى ثلاث طبقات رئيسية:
*   **Model:** يحتوي على الكلاسات التي تمثل كيانات البيانات الأساسية (مثل `Student`, `Teacher`, `Grade`, `User`, `Transaction`).
*   **View:** واجهات المستخدم المصممة باستخدام ملفات `FXML` وبتنسيق CSS لإضفاء مظهر عصري.
*   **Controller:** الكود البرمجي الذي يربط بين البيانات والواجهات ويعالج العمليات المنطقية والأحداث.

### 2. مفاهيم OOP المُطبقة
*   **التجريد (Abstraction):** تمثيل الكيانات الأساسية عبر كلاسات مجردة مثل `Person` و `User`.
*   **الوراثة (Inheritance):** إعادة استخدام الكود عبر توريث الخصائص المشتركة من `Person` إلى كلاسات `Student` و `Teacher`.
*   **التغليف (Encapsulation):** حماية المتغيرات داخل الكلاسات وجعلها ذات وصول خاص (`private`) مع إمكانية قراءتها وتعديلها عبر دوال `getters/setters`.
*   **تعدد الأشكال (Polymorphism):** استخدام واجهة `Payable` لحساب رواتب المعلمين بصيغة ديناميكية تختلف بحسب طبيعة التعاقد.

### 3. أنماط التصميم (Design Patterns)
*   **Singleton Pattern:** تم تطبيقه في كلاس `DatabaseConnection` لضمان وجود اتصال فريد وموحد بقاعدة البيانات طوال فترة عمل التطبيق.
*   **Strategy Pattern:** عبر الـ Interface `Payable` لتغيير سلوك وطريقة حساب الأجور ديناميكياً للمعلمين.
*   **Session & RBAC Pattern:** لتتبع المستخدم النشط والتحقق من صلاحياته في كل شاشة.

## ⚙️ خطوات التثبيت والتشغيل (Installation & Running)

### المتطلبات المسبقة:
1.  تثبيت حزمة تطوير جافا **JDK 17** أو أعلى.
2.  تثبيت أداة البناء **Maven** (أو استخدام الـ Maven Wrapper المدمج في المشروع).

### خطوات تشغيل التطبيق:

1.  **تحميل المشروع:**
    ```bash
    git clone https://github.com/your-username/EduFinance.git
    cd EduFinance
    ```

2.  **تنظيف وبناء المشروع (Build Project):**
    ```bash
    mvn clean install
    ```

3.  **تشغيل التطبيق (Run Application):**
    ```bash
    mvn javafx:run
    ```

> [!NOTE]
> عند التشغيل لأول مرة، سيقوم التطبيق تلقائياً بإنشاء ملف قاعدة البيانات المحلي `edufinance.db` وتغذية الجداول الافتراضية والبيانات التجريبية تلقائياً دون الحاجة لأي تهيئة يدوية.
